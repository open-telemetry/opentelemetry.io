Цей розділ містить документацію, що описує робочі процеси CI/CD, скрипти NPM, і допоміжні скрипти, які використовуються для збирання сайту OpenTelemetry та для його обслуговування.

## Типи збірки: повна та скорочена {#build-kinds}

На додачу до **повної** (звичайної) збірки сайту Hugo, Docsy підтримує **скорочені збірки**, які, серед іншого, дозволяють набагато швидше перевіряти посилання, зберігаючи повне покриття перевірки. Детальніше дивіться [Chrome build modes][] у документації Docsy.

Деякі npm-скрипти збірки завжди будують той самий тип сайту (повний або скорочений). Інші використовують значення змінної середовища `BUILD_KIND` і зазвичай мають значення `lean`, якщо вона не встановлена.

| Скрипт                     | Тип збірки   | Drafts/future | Мініфікація |
| -------------------------- | ------------ | ------------- | ----------- |
| `build`                    | `BUILD_KIND` | так           | ні          |
| `build:full`               | full         | так           | ні          |
| `build:lean`               | lean         | так           | ні          |
| `build:preview`            | full         | так           | так         |
| `build:production`         | full         | ні            | так         |
| `log:build` (артефакт CI)  | `BUILD_KIND` | так           | ні          |
| `netlify-build:preview`    | full         | так           | так         |
| `netlify-build:production` | full         | ні            | так         |
| Більшість інших команд     | `BUILD_KIND` | так           | ні          |

Більшість перевірок, які спочатку виконують чисту збірку, наприклад перевірка посилань, використовують `BUILD_KIND`. Детальніше про скрипти перевірки посилань дивіться в [Перевірка посилань](./link-checking/).

Якщо вам цікавий розмір та продуктивність різних типів збірки, виконайте `npm run metrics:build` (повна) або `npm run metrics:build -- --lean`.
